# 错误码与响应格式

## 请求格式

- **Content-Type**: `application/json`

---

## 响应格式

```typescript
interface ApiResponse<T> {
  code: number
  message: string
  data: T
}
```

---

## HTTP 状态码

| 状态码 | 含义 | 处理建议 |
|--------|------|----------|
| 200 | 成功 | 正常处理 |
| 400 | 请求参数错误 | 检查请求参数 |
| 401 | 未授权 | 检查认证信息 |
| 403 | 禁止访问 | 检查权限 |
| 404 | 资源不存在 | 检查请求路径 |
| 500 | 服务器内部错误 | 稍后重试 |

---

## 业务错误码

错误码格式：`[模块前缀]-[错误类型]-[序号]`

| 错误码 | HTTP 状态码 | 错误信息 | 触发条件 | 处理建议 |
|--------|------------|----------|----------|----------|
| <!-- USER-VAL-001 --> | <!-- 400 --> | <!-- 用户名不能为空 --> | <!-- 创建用户时用户名为空 --> | <!-- 填写用户名 --> |

### 错误类型编码

| 类型编码 | 含义 |
|----------|------|
| VAL | 参数校验错误 |
| BIZ | 业务规则违反 |
| AUTH | 认证/授权错误 |
| SYS | 系统内部错误 |
