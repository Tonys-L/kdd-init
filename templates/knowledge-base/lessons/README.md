## 写作规范

每条教训包含以下字段：

```text
### N.M 标题

**问题**: 简述遇到的现象
**原因**: 根因分析
**解决方案**: 具体修复方式（含代码示例）
**影响文件**: 涉及的源码文件
**日期**: YYYY-MM-DD
```

新增教训时：
1. 按业务分类归入对应文件（如无对应文件则新建，文件名以业务分类命名）
2. 编号接续当前最大编号
3. 更新本索引的条目数和最近更新日期
4. 填写业务分类（对应 boundaries.md 中的能力名称；如 boundaries.md 不存在，按模块职责归类）
